# ADR 流程

ADR 用于记录长期有效的工程架构和治理决策。它描述决策当时的上下文、选择、结果和被放弃的替代方案。

## 使用规则

1. 从 [template.md](template.md) 复制新 ADR。
2. 文件名使用 `NNNN-short-title.md`，编号递增。
3. ADR 一旦接受，不直接改写历史决策；如需改变，应新增 ADR 并标记替代关系。
4. 与规范相关的 ADR 应在规范文档或 RFC 中建立链接。

## 状态

ADR 状态可使用：

1. `proposed`：提议中。
2. `accepted`：已接受。
3. `deprecated`：已弃用。
4. `superseded`：已被替代。
