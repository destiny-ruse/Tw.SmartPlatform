# 前端样式规范

## 目标

前端样式必须在视觉一致性、响应式布局和主题扩展之间保持清晰边界，避免页面级样式互相污染。统一样式规则可以降低重复变量、不可维护选择器和移动端布局溢出的风险。

## 适用范围

适用于 Vue 组件样式、全局样式、设计变量、主题变量、响应式断点、图标和静态资源引用、样式构建配置以及视觉验收说明。第三方库原始样式不属于本规范范围；对第三方组件进行项目封装或覆盖时必须遵守本规范。

## 规则

1. 颜色、字号、间距、层级、圆角和阴影等可复用值必须来自设计变量或项目样式 token，不得在多个页面重复硬编码。
2. 组件私有样式必须限制作用域，公共样式必须集中维护并具备明确命名，不得用高优先级选择器覆盖未知范围。
3. 响应式布局必须使用稳定的断点、弹性容器或栅格规则，不得只为单一屏宽写死像素宽度。
4. 样式不得依赖 DOM 深层结构或第三方组件内部类名，除非封装层明确承担该适配责任。
5. 主题变量和业务状态样式必须分层，组件不得直接改写全局主题变量来表达局部状态。
6. 可访问性相关样式必须保留焦点、错误、禁用、加载和高对比状态，不得只提供默认视觉状态。
7. 静态资源引用必须使用项目约定路径和命名，构建后资源大小、格式和缓存影响应当在大资源变更中说明。
8. 新增样式应当验证桌面、窄屏、长文本、国际化文本和数据为空时的布局，不得只验证默认中文短文本。

## 示例

正例：

```scss
.order-card {
  padding: var(--spacing-md);
  color: var(--color-text-primary);
  background: var(--color-surface);
}

@media (max-width: 640px) {
  .order-card {
    display: grid;
    gap: var(--spacing-sm);
  }
}
```

反例：

```scss
.page .ant-card .ant-card-body div div {
  padding: 17px !important;
  color: #333333;
}
```

## 检查清单

- 可复用视觉值是否来自变量或 token。
- 组件私有样式和公共样式边界是否清晰。
- 响应式布局是否覆盖窄屏、长文本和空状态。
- 是否避免深层选择器、全局污染和过度 `!important`。
- 焦点、错误、禁用、加载和高对比状态是否保留。
- 静态资源变更是否说明格式、大小或缓存影响。
