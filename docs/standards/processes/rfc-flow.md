# RFC 流程规范

<!-- anchor: goal -->
## 目标

让跨团队或高风险工程变更在实施前形成可讨论、可追踪的方案。

<!-- anchor: scope -->
## 适用范围

适用于架构调整、公共契约变化、基础设施变更和规范体系演进。

<!-- anchor: flow -->
## 流程

1. 提出背景和目标。
2. 列出备选方案和取舍。
3. 邀请受影响角色评审。
4. 记录结论并跟踪实施。

<!-- anchor: template -->
## 模板

RFC 至少包含背景、目标、非目标、方案、影响范围、迁移方式和验收标准。

<!-- anchor: checklist -->
## 检查清单

- 是否明确非目标。
- 是否列出受影响系统。
- 是否说明回滚或迁移方式。
