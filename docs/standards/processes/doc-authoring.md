# 文档撰写流程

<!-- anchor: goal -->
## 目标

文档撰写流程用于让跨团队工程活动具备明确入口、责任人和验收结果。

<!-- anchor: scope -->
## 适用范围

适用于标准文档、依赖引入、工具配置和工程流程变更。

<!-- anchor: flow -->
## 流程

1. 说明背景和目标。
2. 识别受影响系统和角色。
3. 完成评审和验证。
4. 记录结论、版本和后续责任。

<!-- anchor: rules -->
## 规则

流程中的关键结论必须可追踪；例外必须说明原因和到期时间。

<!-- anchor: checklist -->
## 检查清单

- 是否明确输入和输出。
- 是否完成必要评审。
- 是否记录验证命令或验收证据。
