# RFC 流程

RFC 用于讨论新增规范、重大规范变更或会影响多个团队的工程治理调整。

## 使用规则

1. 从 [template.md](template.md) 复制新 RFC。
2. 文件名使用 `YYYYMMDD-short-title.md`。
3. RFC 必须说明问题、目标、备选方案、推荐方案、影响范围和迁移方式。
4. RFC 通过后，再更新规范文档、机器规则和索引。

## 状态

RFC 状态可使用：

1. `draft`：草稿。
2. `review`：评审中。
3. `accepted`：已接受。
4. `rejected`：已拒绝。
5. `superseded`：已被替代。
